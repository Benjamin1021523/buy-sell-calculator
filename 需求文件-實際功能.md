# 技術

* 前端專案，使用 Vue 3 + Vite + Vue Router
* 使用 GitHub Actions 在 push 時 build 並部署至 GitHub Pages
* 建置時複製 `index.html` 為 `404.html`，讓直接訪問 `/buy`、`/sell` 等路徑時仍能正確載入 SPA

# 需求內容

目的：製作單頁網頁（不會呼叫外部 API），使用者在頁面輸入資訊進行計算

功能頁面分成 `買進計算機` 與 `賣出計算機`

使用者可以在股票交易前，藉由此頁面輸入買賣資訊，以此計算最有利的買賣數量

# 頁面導航

* 使用 Tab 切換兩個功能頁面
* 路徑分別為 `/buy`（買進計算機）、`/sell`（賣出計算機）
* 根路徑 `/` 自動導向 `/buy`

# 通用名詞定義

## 成交金額

* 股價 × 股數，小數點捨去
* 計算時加上微小值避免浮點數誤差

## 手續費

* 買賣皆收取，成交金額的 `0.1425%`，小數點捨去，最少為 1

## 交易稅

* 僅賣出時收取，成交金額的 `0.3%`，小數點捨去，最少為 0
* 依據交易商品不同，有的免交易稅

## 最大成交金額

* 計算時的系統限制，最高 10000 元，用以避免無限制向上嘗試

# 買進計算機（`/buy`）

## 買進規則

* 買進花費：買進 `成交金額` + `手續費`
* 以手續費 1 元為目標，若 1 股就使手續費超過 1 元，則以 1 股為最終答案
* 目標是得到最低的平均成本

## 畫面構成

* 輸入框：
  * 買進目標價格（ex: 9.7）
* 輸出：
  * 買進花費（整數）
  * 成本最低買進數量（整數，單位：股）
  * 買進手續費金額（整數）
  * 此時平均成本（元/股，最多顯示到小數點 10 位）

---

# 賣出計算機（`/sell`）

## 賣出規則

* 賣出收益：賣出 `成交金額` - `手續費` - `交易稅`
* 目標是得到最高的平均收益

## 畫面構成

* 輸入框：
  * 賣出目標價格（ex: 9.7）
  * 有無交易稅（勾選項目，勾選 `無交易稅` 則不計算交易稅；預設勾選）
* 輸出：
  * 賣出所得（整數）
  * 最高收益賣出數量（整數，單位：股）
  * 賣出手續費金額（整數）
  * 交易稅金額（整數）
  * 此時平均收益（元/股）

---

# 手續費折扣

* 頁面右上角可輸入手續費折扣（格式：XX折）
* 買進與賣出的手續費皆依此折扣計算
* 未設定時預設為 10折（無折扣）；設定值存入 localStorage，下次開啟時自動載入
* 解析規則：1 位數除 10、2 位數除 100；10、100 折視為無折扣

# 配色切換

* 頁面左上角有「配色切換」標籤及 A、B、C 三個按鈕
* 點擊按鈕即可切換三種介面配色
* 預設為 A 配色；設定值存入 localStorage，下次開啟時自動載入
* 按鈕顯示規則：
  * 被選擇的樣式：方塊外顯示厚邊框，按鈕底色與文字顏色與該樣式配色相同
  * 游標移至未選擇的按鈕時：顯示該樣式的配色
* 三種配色：
  * A：灰藍中性
  * B：暖色琥珀
  * C：綠色

# 系統參數說明

* 頁面最下方固定顯示（不隨買進/賣出版面變化而上下移動）
* 說明內容：本頁面以手續費比率 0.1425%（買賣皆收取，小數點捨去，最低 1 元）、交易稅比率 0.3%（僅賣出時收取，小數點捨去，最低 0 元）、最大成交金額 10,000 元為計算參數。手續費可於右上角輸入折扣。
