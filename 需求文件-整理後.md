# 技術

* 前端專案，使用 Vue 3
* 預計使用 GitHub Actions 在 push 時 build 並部署到另一分支，以此發布 GitHub Pages，完成後請提供設置文件

# 需求內容

目的：製作單頁網頁（不會呼叫外部 API），使用者在頁面輸入資訊進行計算

功能頁面分成 `買進計算機` 與 `賣出計算機`

使用者可以在股票交易前，藉由此頁面輸入買賣資訊，以此計算最有利的買賣數量

# 頁面導航

* 使用 Tab 切換兩個功能頁面
* 路徑分別為 `/buy`（買進計算機）、`/sell`（賣出計算機）

# 通用名詞定義

## 成交金額

* 股價 × 股數，小數點捨去

## 手續費

* 買賣皆收取，成交金額的 `0.1425%`，小數點捨去，最少為 1

## 交易稅

* 僅賣出時收取，成交金額的 `0.3%`，小數點捨去，最少為 0
* 依據交易商品不同，有的免交易稅

## 最大成交金額

* 計算時的系統限制，最高 10000 元，用以避免無限制向上嘗試

# 買進計算機（`/buy`）

## 買進規則

* 買進花費：買進 `成交金額` + `手續費`
* 以手續費 1 元為目標，若 1 股就使手續費超過 1 元，則以 1 股為最終答案
* 目標是得到最低的平均成本
* 舉例：計算以 9.7 元買進，買進幾股的成本最低
  * 144 股 → 1396.8 + 1 = 1397，1397/144 即為平均成本
  * 143 股 → 1395.1 + 1 = 1396，1396/143 即為平均成本
  * 以此計算後，買進 144 股的成本較優

## 畫面構成

* 輸入框：
  * 買進目標價格（ex: 9.7）
* 輸出：
  * 買進花費（整數，ex: 1397）
  * 成本最低買進數量（整數，ex: 144）
  * 買進手續費金額
  * 此時平均成本（股/元，ex: 9.70138889，最多顯示到小數點 10 位）

---

# 賣出計算機（`/sell`）

## 賣出規則

* 賣出收益：賣出 `成交金額` - `手續費` - `交易稅`
* 目標是得到最高的平均收益
* 舉例：計算以 9.7 元賣出，在沒有交易稅的情況，賣出幾股的收益最高
  * 144 股 → 1396.8 - 1 - 0（交易稅 0）= 1395，1395/144 即為平均收益
  * 143 股 → 1395.1 - 1 - 0（交易稅 0）= 1394，1394/143 即為平均收益
  * 以此計算後，賣出 143 股的收益較優

## 畫面構成

* 輸入框：
  * 賣出目標價格（ex: 9.7）
  * 有無交易稅（勾選項目，勾選 `無交易稅` 則不計算交易稅）
* 輸出：
  * 賣出所得（整數）
  * 最高收益賣出數量（整數，ex: 143）
  * 賣出手續費金額、交易稅金額
  * 此時平均收益（股/元）

# 產生畫面規則

* 在顯示內容相同的前提下，請產生 3 種風格的頁面，並透過頁面左上角的「配色切換」按鈕切換顯示風格
* 按鈕 A、B、C 分別對應三種配色
* 在驗收確認畫面後會由用戶決定採用哪種風格，再轉換為固定其中一種畫面
