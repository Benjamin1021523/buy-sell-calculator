# 買賣計算機 - 設置文件

本文件說明專案建置、部署與驗收相關設定。

---

## 一、開發環境設置

### 必要環境

- Node.js 18 以上
- npm

### 安裝與啟動

```bash
npm install
npm run dev
```

開發伺服器預設：`http://localhost:5173`

---

## 二、GitHub Pages 部署

### 1. 前置條件

- 專案已推送到 GitHub
- 倉庫為 public 或具 Pages 權限

### 2. 啟用 GitHub Pages

1. 進入倉庫 **Settings** → **Pages**
2. **Build and deployment** → **Source** 選擇 **GitHub Actions**

### 3. 工作流程說明

推送至 `main` 或 `master` 時，`.github/workflows/deploy.yml` 會：

1. 安裝依賴
2. 執行 `npm run build`
3. 複製 `index.html` 為 `404.html`（讓直接訪問 /buy、/sell 等路徑時，GitHub Pages 以 404 頁載入 SPA，Vue Router 才能正確處理）
4. 將 `dist/` 部署至 GitHub Pages

### 4. base 路徑設定

| 部署類型 | 網址範例 | VITE_BASE 設定 |
|----------|----------|----------------|
| 專案站點 | `https://user.github.io/買賣計算機/` | `/買賣計算機/`（或使用 workflow 預設） |
| 根站點 | `https://user.github.io/` | `/` |

若需修改，請編輯 `.github/workflows/deploy.yml` 的 `VITE_BASE` 環境變數。

---

## 三、配色切換

點擊頁面左上角「配色切換」旁的 A、B、C 按鈕，即可切換三種介面配色。

驗收後若需固定風格，可移除按鈕並在程式中寫死對應風格。

---

## 四、驗收項目

### 買進計算機（/buy）

- [ ] 輸入買進目標價格後，顯示買進花費、成本最低買進數量、此時平均成本
- [ ] 例：9.7 元 → 買進花費 1397、144 股、平均成本約 9.70138889

### 賣出計算機（/sell）

- [ ] 輸入賣出目標價格、勾選/取消「無交易稅」
- [ ] 顯示賣出所得、最高收益賣出數量、手續費、交易稅、此時平均收益
- [ ] 例：9.7 元、無交易稅 → 143 股、賣出所得 1394

### 共通

- [ ] Tab 可切換買進/賣出
- [ ] 左上角「配色切換」按鈕 A、B、C 可切換三種配色
- [ ] 最大成交金額限制 10000 元
